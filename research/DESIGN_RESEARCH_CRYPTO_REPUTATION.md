# TrustTap+ Design Research: Crypto Reputation & Trust Scoring Landing Pages
**Date:** March 1, 2026
**Research Focus:** Top DeFi/crypto reputation, trust scoring, and analytics platform design patterns (2025-2026)

---

## Executive Summary

Based on research of Awwwards, design galleries, and ecosystem analysis, modern crypto reputation/trust platforms share these visual patterns:

**What They ALL Do (Differentiation Imperative for TrustTap+):**
- Purple/violet + teal/cyan as primary ecosystem colors (Solana standard)
- Dark mode as default (reduces eye strain, amplifies data viz)
- Complex gradient backgrounds with kinetic motion
- Generic card-based layouts (title + icon + subtitle = "AI slop")
- Heavy reliance on real-time data visualization dashboards
- 3D/immersive elements for NFT or token showcase

**What TrustTap+ Must Avoid:**
- Purple as accent (ecosystems use #14F195 or #9945FF)
- Glass morphism (overused in 2024-2025, feels dated)
- Stock gradient backgrounds
- Dashboard drowning in charts and metrics
- Overly technical jargon in hero messaging

## Part 2: Solana Ecosystem Visual Pattern Analysis

### **What ALL Solana Ecosystem Products Share**

| Pattern | Examples | Hex Values |
|---------|----------|-----------|
| **Primary Color** | Phantom, Magic Eden, Jupiter, Solana Brand | `#9945FF` (Solana Purple) |
| **Secondary Accent** | Magic Eden, Serum, Raydium | `#14F195` (Solana Teal) |
| **Background** | Phantom, Jupiter, Magic Eden | `#000000` or `#0F0F0F` (Pure Black) |
| **Tertiary Accent** | Some gradient uses | `#00D4FF` (Cyan) |
| **Text Primary** | All platforms | `#FFFFFF` (White) |
| **Text Secondary** | All platforms | `#A0AEC0` or `#718096` (Gray-400 to Gray-500) |

### **Design Language Characteristics**

1. **"Speed & Kinesis" Metaphor**
   - Angular, geometric typography (1980s tech revival)
   - Gradient overlays implying motion
   - Smooth animations showing throughput
   - Minimal hierarchy (everything feels urgent)

2. **Dark Mode as Default**
   - Not an option — the standard
   - Reduces eye strain during extended trading
   - Amplifies data visualization and status indicators
   - Conveys "cutting-edge sophistication"

3. **Minimal Hierarchy, Maximum Data**
   - Card-based layouts dominate
   - Charts/metrics take visual priority over prose
   - Copy reduced to essentials + CTAs
   - Wallets/addresses truncated (first/last 4 chars)

4. **Generic Card Patterns** (The "AI Slop" Problem)
   - Icon + Title + Subtitle = Default component
   - No unique visual language
   - Copy-pasted across projects
   - Feels mass-produced

5. **Transaction-Focused UI**
   - Confirmation previews & safeguards
   - Progressive disclosure (hide complexity until needed)
   - State indicators (pending/confirmed/failed)
   - Fee estimation always visible

## Part 3: Web3/Crypto Trust Design Principles (Emerging 2025-2026)

### **Why Design Matters in Web3**

From Coinbound research: "Web3 design has two jobs: help people do things, and help them believe they're safe doing it."

**Key Finding:** Exceptional design builds the trust required for adoption. Poor Web3 design actively hinders mainstream use.

### **10 Patterns That Build Trust**

1. **Progressive Onboarding** — Introduce blockchain gradually using familiar language
2. **Transaction Previews** — Show amounts, fees, recipient before execution
3. **Visual State Indicators** — Progress bars, spinners, clear error messages
4. **Secure Defaults** — Minimal permission scopes, toggle switches for authorization
5. **Identity & Reputation Signals** — Verified badges, audit confirmations, community metrics
6. **Graceful Recovery** — Undo windows, block explorer links, support options on failure
7. **Lifecycle Notifications** — Multi-channel alerts throughout transaction completion
8. **Invisible Complexity** — Hide mechanics unless needed; contextual sandboxes for confirmation
9. **Mobile-First Optimization** — Biometric auth, gesture confirmation, responsive layouts
10. **Accessibility & Localization** — Screen reader support, scalable fonts, native-language interfaces

### **Reputation Scoring UI Patterns** (Emerging)

From TrustScores research:

- **Status Gauges:** Large, prominent trust/reputation score circles with color coding
- **Distribution Summaries:** High-score vs. low-score ratio (red/yellow/green)
- **Trend Visualizations:** Sparklines showing score trajectory (up/stable/down)
- **Real-Time Updates:** Auto-refresh, no polling delays
- **Drill-Down Capability:** Click metrics to reveal underlying data
- **Comparative Context:** Score vs. peer average, benchmark against similar wallets
- **Clear Grading:** Colors matter (red=risk, yellow=caution, green=trusted)

## Part 4: Design Techniques to Memorize (Top 5 Platforms)

### **Chainalysis: Interactive Flow Visualization**
**Why Memorable:** Animates blockchain complexity into understandable movement
- Nodes represent addresses/transactions
- Lines show fund flow with directional arrows
- Pulsing animations indicate "hot" addresses
- Color coding (red/orange/green) shows risk levels
- **Takeaway:** Abstraction through motion makes technical data accessible

### **Phantom: Native Mobile Integration**
**Why Memorable:** Doesn't fight form factor — embraces it
- Simple balance as hero content (not a metric card)
- Gesture-based navigation (native bottom tabs)
- NFT gallery feels integrated (not bolted-on)
- Biometric auth is primary, not secondary
- **Takeaway:** Mobile-first means rethinking priorities, not responsive cards

### **Jupiter: Swap Route Transparency**
**Why Memorable:** Makes invisible backend visible to user
- Shows exact pool routing before swap execution
- Slippage impact highlighted in warnings
- Route animation shows path taken after swap
- Advanced options collapse/expand seamlessly
- **Takeaway:** Trust through transparency; show the work

### **Magic Eden: Parallax Grid + Hover Transform**
**Why Memorable:** Creates depth and excitement at browse time
- NFT images parallax scroll while cards stay fixed
- Hover swaps between floor price and rarity rank
- Countdown timers on featured drops create urgency
- Verified creator badges are micro-interactions
- **Takeaway:** Subtle motion creates engagement without distraction

### **Scorechain: Wallet Clustering + Risk Mapping**
**Why Memorable:** Simplifies compliance complexity visually
- Related wallets cluster together in graph view
- Risk scores color-code clusters (red/yellow/green)
- Timeline shows transaction flow progression
- Report generation is visible process (not black box)
- **Takeaway:** Show the analysis; don't hide behind a score

## Part 5: What TrustTap+ Must Differentiate On

### **DO NOT:**
- ❌ Use purple as primary/accent (ecosystem owns it)
- ❌ Generic "icon + title + subtitle" card layouts
- ❌ Gradient backgrounds without meaning/motion
- ❌ Glass morphism (2024-2025 trend, feels dated)
- ❌ Confusing technical jargon in hero copy
- ❌ Dashboard drowning in charts (information overload)
- ❌ Dark mode as assumption (even Phantom does it)

### **DO:**
- ✅ Use **emerald/teal** as primary (differentiates from Solana purple ecosystem)
- ✅ Design unique visual language for reputation scoring (not generic cards)
- ✅ Create **memorable hero moment** that communicates trust instantly
- ✅ Embrace **simplicity over data density** — focus on ONE key metric (trust score)
- ✅ Use **motion strategically** — only to clarify, not to impress
- ✅ Design for **Seeker phone context** — mobile-native, not web responsive
- ✅ Make **reputation transparent** — show how score is calculated
- ✅ Build **social proof** visually — show verified SGT owners, social connections

### **Differentiation Opportunities**

1. **One-Metric Hero** — Instead of dashboard overload, ONE trust score as centerpiece
   - Emerald color scheme (not purple)
   - Single gauge/ring visualization
   - Micro-copy explaining score instantly

2. **Social Reputation** — Unlike Chainalysis (institutional) or Phantom (wallet-centric)
   - Show your verified connections (SGT holders you've met)
   - Visual network graph of trust relationships
   - Community trust signals (not just on-chain metrics)

3. **Meeting as Verification** — Unique to TrustTap+
   - QR/NFC connection creates permanent record
   - Verified connection badge in profile
   - Trust score includes social proof (not just transaction history)

4. **Mobile-Native, Not Web-Responsive**
   - Gesture-based interactions (swipe, long-press)
   - Bottom-sheet modals for deep dives
   - Biometric auth for sensitive operations
   - Native Android optimizations (not PWA band-aid)

5. **Sybil-Resistant Scoring**
   - SGT gating proves human verification
   - Meeting history visible (not just score)
   - Community trust multiplier (SGT holders trust each other more)
   - Transparent algorithm (how score is calculated)

| **Neutral** | Slate-700 | `#334155` | Borders, dividers |

### **Typography Stack**

| Element | Font | Size | Weight | Line-Height |
|---------|------|------|--------|-------------|
| **Display** | System sans-serif (Inter preferred) | 32-48px | 700 | 1.2 |
| **Headline (H1-H3)** | System sans-serif | 24-32px | 600-700 | 1.2-1.3 |
| **Body** | System sans-serif | 16px | 400 | 1.5 |
| **Body Small** | System sans-serif | 14px | 400 | 1.5 |
| **Monospace (Addresses)** | SF Mono, Monaco, monospace | 12-14px | 400 | 1.4 |
| **Numeric Emphasis** | System sans-serif | 18-28px | 600 | 1.2 |

**Key Rules:**
- No thin weights (< 400) — readability on mobile
- Large line-height (1.5+) — reduces cognitive load
- Monospace ONLY for wallet addresses and raw data
- Numeric emphasis for trust scores, balances, metrics

## Part 7: Hero Section Patterns to Avoid

### **What Doesn't Work:**
1. **Gradient Background Hero** — Overused, feels generic
2. **Video Background with CTA Overlay** — Bandwidth issue on mobile, outdated
3. **Floating 3D Objects** — Impressive but distracting, no conversion benefit
4. **Carousel of Features** — Too much to process, users bounce
5. **Stock Illustration + Copy** — Generic, lacks trust

### **What Works for Trust Platforms:**
1. **Hero = One Key Metric** (Trust Score, Reputation Badge)
   - Large, prominent display
   - Clear sub-copy explaining what it means
   - Single CTA (Connect Wallet / Get Scored)
   - Minimal animation (maybe subtle glow or pulse)

2. **Hero = Social Proof**
   - Verified user count (live counter)
   - Recent connections (avatars/names scrolling)
   - Community trust indicators
   - Testimonial snippet

3. **Hero = Clear Value Prop**
   - Headline (6-10 words max)
   - Sub-headline (one sentence explaining benefit)
   - Visual: icon or simple illustration
   - CTA: clear call to action
